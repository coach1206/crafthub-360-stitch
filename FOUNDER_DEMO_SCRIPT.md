# Founder Demo Script — NOVEE OS

> Phase 13 · Founder / Investor / Pilot Package  
> Route: `/founder-demo` — Founder Level 0 only

---

## Before You Start

- Log in via `/founder-login` with your Founder PIN
- Navigate to `/founder-demo`
- Start a demo session and select the appropriate demo type and audience
- Keep the Founder Demo Panel open in a second tab for talking points
- Have the system running on a tablet in landscape orientation

---

## 5-Minute Version

**Total target: 5 minutes. Move with confidence. Do not over-explain.**

| Time | Section | What to Do |
|------|---------|------------|
| 0:00 | What NOVEE OS Is | One sentence. "NOVEE OS is a guest intelligence platform for premium hospitality." Point at the screen. |
| 0:30 | Guest Journey | Walk to the home screen. "A guest walks up, taps the screen." Navigate to SmokeCraft. |
| 1:15 | SmokeCraft | Choose a mentor, start the session. "They choose a mentor who matches their taste. The system guides them." Let it run for 45 seconds. |
| 2:00 | Passport | Complete or skip to the passport stamp. "They earn a stamp. Every visit builds a relationship." Show the leaderboard. |
| 2:30 | POS 3 | Navigate to `/pos`. "Staff see the guest's session in real time. They know what to recommend before the guest even approaches." |
| 3:00 | E.A.T. | Navigate to `/eat`. "Managers see what's working across the whole floor tonight." Point at one feed. |
| 3:30 | Kiosk | "This runs on any iPad. No app store. No proprietary hardware." |
| 4:00 | Venue Testing | Navigate to `/venue-test`. "Before any venue goes live, we run a structured readiness test. The score tells us exactly what's ready and what isn't." Show the score. |
| 4:30 | The Ask | Return to the room. "We're looking for [X] pilot venues. One tablet. 30 minutes with your staff. Real guest data from day one." |

## 15-Minute Version

**Add these sections after the 5-minute flow:**

### Mentor Voice (add 2 minutes)
- Complete a full SmokeCraft session with audio on
- "The voice is generated by AI — but it adapts to the mentor's character, not a generic script"
- Toggle mute mode: "If it's loud in the venue, guests can read along"

### Passport Deep Dive (add 2 minutes)
- Show the leaderboard: "The top guest here has 14 stamps — each one is a visit, a conversation, a relationship"
- "This is not a punch card. It's behavioral data — what they tasted, what they chose to learn about"

### E.A.T. Command Deep Dive (add 3 minutes)
- Walk through Environment, Asset, Transaction panels
- "The question every manager asks is: what's working tonight? This answers it"
- Show the session activity feed

### Pilot Readiness (add 3 minutes)
- Navigate to `/venue-test`
- Show the readiness score: "This is what we require before any venue goes guest-facing"
- Show the issue log: "Every friction point is logged. Every session is scored."

## What NOT to Over-Explain

- Do NOT spend more than 20 seconds on the tech stack
- Do NOT explain the database schema
- Do NOT demo the admin panel or founder controls to non-founder audiences
- Do NOT claim live revenue numbers — say "pilot-stage" and move on
- Do NOT explain what prototype mode is unless asked — then use the script below

### If Someone Asks About Prototype Mode

> "What you're seeing is a fully functional system running on simulated POS data. In a live venue, POS 3 and E.A.T. connect to the venue's actual point-of-sale. The guest experience — SmokeCraft, the voice, the Passport — is exactly what goes in front of a real guest."

---

## What to Do If Something Breaks

- If a page is blank: refresh once, then say "We're running on a live dev environment — give me 10 seconds" and navigate home
- If voice doesn't load: toggle to mute/read mode — "The system has a read mode for loud environments, which we'd use here anyway"
- If the demo resets unexpectedly: navigate to `/founder-demo` and start a new session — prototype data is regenerated instantly

---

## Demo Reset Safety

From `/founder-demo` → **Launch tab** → **Reset Demo Data**

This safely clears:
- Active demo sessions and events
- Prototype guest passport data
- Venue test demo state (if selected)

It does NOT clear:
- Auth credentials
- Founder controls
- Audit logs
- User accounts
